# 量潮创始人官网 — 设计

## 定位

创始人官网。展示我对外交付的成果——产品、文字、作品。成果本身可以包含改稿作品，但不展示改稿过程。

核心问题只有一个：**这个人交付过什么？**

## 结构

### 首页 — Hero

一句话说明我在做什么、交付什么。不加多余信息。

> 创业者，写作者。
> 已交付的产品与文字，定义了我的认知。

### 关于

简要背景：做过什么、能做什么、关注什么。不超过三段话。不写"在探索"——只写已确认的。

### 交付

已发布的成果。每一项包括：
- 名称 + 链接
- 一句话说明是什么
- 发布时间

已交付：
- **量潮** — 知识工程公司，已发布产品 X
- **认知-叙事系列** — 已发表的系列文章
- **小说改稿作品** — 改稿完成的作品

### 作品（首页 section）

首页只展示最近 3 条作品，附带 "查看全部 →" 链接到 `/works`。

每项包括：
- 标题（链接到作品原文）
- 类型标签：改稿 / 文章 / 其他
- 日期
- 一句话说明

### 联系

- GitHub
- 邮件
- 其他社交链接

不需要表单。

## 作品页面 `/works`

### 路由

`/works` — 独立页面，从首页 "查看全部" 进入。顶部包含返回首页的链接。

### 列表

按时间倒序排列。每项：

```
[类型] 标题              日期
       一句话说明
```

示例：

```
[改稿] 黄昏的颜色         2026-05
       完成了某篇小说的整体叙事调整和语言润色。

[文章] 认知与叙事的一致性  2026-04
       探讨写作中认知框架对叙事结构的影响。
```

类型标签用灰色文字，不加背景色、边框、badge。

### 筛选（可选）

页面顶部提供三个文字链接切换视图：`全部 / 改稿 / 文章`。选中状态为加粗，其余正常字重。无选中则显示全部。

### 空状态

筛选后无结果：显示 `暂无匹配作品`，灰色文字。

### 作品详情（暂不设计）

当前仅列表页。每项链接直接跳到作品原文（外部链接）。未来如需为每个作品写说明页再补充。

## 风格原则

### 克制

- 不加多余的视觉元素。没有动画、没有渐变背景、没有卡片阴影。
- 颜色只用黑色、白色、灰色。
- 字体：sans-serif，系统字体栈。

### 文字优先

- 页面内容就是文字。不加图表、插图、icon。
- 唯一允许的非文字元素是项目链接。
- 排版是唯一的视觉设计手段：字号、字重、行距、间距。

### 留白

- 每个 section 上下 5rem padding。
- 行距 1.8。
- 页面宽度 720px。

## 技术决策

- React 19 + TypeScript + Vite
- 纯 CSS，无框架
- 部署到 GitHub Pages
- 无后端，纯静态
