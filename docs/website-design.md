# 量潮创始人官网 — 设计

## 定位

创始人官网。展示我对外交付的成果——产品、文字、作品。成果本身可以包含改稿作品，但不展示改稿过程。

核心问题只有一个：**这个人交付过什么？**

## 结构

### 首页 — Hero

一句话说明我在做什么、交付什么。不加多余信息。

> 创业者，写作者。
> 已交付的产品与文字，定义了我的认知。

### 关于

简要背景：做过什么、能做什么、关注什么。不超过三段话。不写"在探索"——只写已确认的。

### 交付

已发布的成果。每一项包括：
- 名称 + 链接
- 一句话说明是什么
- 发布时间

已交付：
- **量潮** — 知识工程公司，已发布产品 X
- **认知-叙事系列** — 已发表的系列文章
- **小说改稿作品** — 改稿完成的作品

### 作品

已公开发表的文字、改稿作品、或其他创作。每项包括标题、日期、链接。只收录完成态的作品，不展示改稿过程记录。

### 联系

- GitHub
- 邮件
- 其他社交链接

不需要表单。

---

## 风格原则

### 克制

- 不加多余的视觉元素。没有动画、没有渐变背景、没有卡片阴影。
- 颜色只用黑色、白色、灰色。
- 字体：sans-serif，系统字体栈。

### 文字优先

- 页面内容就是文字。不加图表、插图、icon。
- 唯一允许的非文字元素是项目链接。
- 排版是唯一的视觉设计手段：字号、字重、行距、间距。

### 留白

- 每个 section 上下 5rem padding。
- 行距 1.8。
- 页面宽度 720px。

---

## 技术决策

- React 19 + TypeScript + Vite
- 纯 CSS，无框架
- 部署到 GitHub Pages
- 无后端，纯静态
