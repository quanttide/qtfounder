# 量潮创始人官网 — 设计

## 定位

创始人官网。不是公司官网，不是博客，不是作品集——是创始人本人的数字存在。

核心问题只有一个：**这个人是谁？**

## 结构

### 首页 — Hero

一句话回答"这个人是谁"。不加多余信息。

> 创业者，写作者。
> 在创造产品与书写叙事之间，寻找认知的一致性。

### 关于

简要背景：做什么的、相信什么、在探索什么。不超过三段话。

### 项目

正在做的事情。每一项包括：
- 名称 + 链接
- 一句话描述
- 时间

当前项目：
- **量潮** — 一家做知识工程的公司
- **qtgame-tycoon / qtgame-war** — 游戏项目
- **认知-叙事实验** — 用 LLM 验证创始人的认知与叙事的一致性

### 写作

公开发布的文字。每篇包括标题、日期、链接。按时间倒序。

### 联系

- GitHub
- 邮件
- 其他社交链接

不需要表单。

---

## 风格原则

### 克制

- 不加多余的视觉元素。没有动画、没有渐变背景、没有卡片阴影。
- 颜色只用黑色、白色、灰色。
- 字体：sans-serif，系统字体栈。

### 文字优先

- 页面内容就是文字。不加图表、插图、icon。
- 唯一允许的非文字元素是项目链接。
- 排版是唯一的视觉设计手段：字号、字重、行距、间距。

### 留白

- 每个 section 上下 5rem padding。
- 行距 1.8。
- 页面宽度 720px。

## 技术决策

- React 19 + TypeScript + Vite
- 纯 CSS，无框架
- 部署到 GitHub Pages
- 无后端，纯静态
